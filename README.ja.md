# YKK スマートコントロールキー — Home Assistant インテグレーション

YKK AP **スマートコントロールキー**（SCK / `com.alpha.lockapp`）BLE
ドアロック用のカスタムインテグレーションです。

プロトコルはリバースエンジニアリングによるものです。YKK APとは無関係です。

## 概要

BLE の役割を2つに分け、それぞれ独立してアダプターを選択できます。

- **長距離（受信専用）** — ロックが発信する暗号化済みステート広告を
  受信し、`lock` エンティティに現在の施錠／解錠状態を反映します。
  併せて電池残量低下、点検時期、ファームウェアバージョン、電波強度
  （RSSI）も診断エンティティとして提供します。広告を受信できれば
  壁越しでも動作するため、HA から見えているスキャナーであれば
  どれでも構いません。
- **近距離（書き込み）** — 施錠／解錠を行うための GATT 通信です。
  ロックから物理的に近いスキャナー（目安 3 m 以内）が必要です。
  通常はドア付近に設置した ESPHome の
  [`bluetooth_proxy`](https://esphome.io/components/bluetooth_proxy.html)
  を使用します。

それぞれの役割でどのスキャナーを使うかは、初期設定フローで指定でき、
インストール後もインテグレーションの **オプション** から変更可能です。

UI は **英語** と **日本語** に対応しています。

## インストール

### HACS（カスタムリポジトリ）

1. HACS → … → *カスタムリポジトリ* を開きます。
2. このリポジトリの URL を **インテグレーション** として追加します。
3. *YKK Smart Control Key* をインストールします。
4. Home Assistant を **再起動** し、新しいインテグレーションを
   読み込みます。
5. *設定 → デバイスとサービス → インテグレーションを追加 → YKK
   Smart Control Key* から追加します。

### 手動インストール

`custom_components/ykk_door/` を Home Assistant の設定ディレクトリの
`custom_components/` 配下にコピーします。HA を再起動し、UI から
インテグレーションを追加してください。

## 初期設定の流れ

1. HA が GATT サービス UUID を見て SCK ロックを自動検出します。
   検出されなかった場合は *インテグレーションを追加 → YKK
   Smart Control Key* で近くの広告を一覧から選べます。
2. **アダプターの選択**：状態取得用（長距離）と操作用（近距離）
   それぞれに使用するスキャナーを選択します。アダプターが1台しか
   ない場合や HA に最適なものを選ばせたい場合は両方とも **自動**
   のままで構いません。ドア付近に ESP32 `bluetooth_proxy` を
   後から導入した際は、インテグレーションの *オプション* から
   再選択できます。
3. **登録**：
   - 近距離アダプターがロック付近にあることを確認します。
   - ロック本体の物理登録ボタンを押し、登録モードに入れます。
   - 6 桁の暗証番号を入力して送信します。
   - GATT による登録ハンドシェイクを実行し、このロック固有の
     広告暗号鍵（`AdvDataKey`）を取得し、設定エントリに保存します。
     この鍵が無いと状態を復号できません。

## エンティティ

| 種類 | 用途 |
|---|---|
| `lock` | 広告から取得した施錠／解錠状態。`lock` / `unlock` サービスは GATT で接続して操作します。 |
| `binary_sensor`（バッテリー） | 電池残量低下フラグ。 |
| `binary_sensor`（problem） | 点検時期フラグ。 |
| `sensor`（diagnostic） | ロック本体およびハンドル部のファームウェアバージョン。 |
| `sensor`（signal_strength） | 長距離スキャナーで観測された最新の RSSI。 |

`lock` エンティティの属性として `long_range_source` /
`short_range_source` / `last_seen_via` も公開しており、現在どの
スキャナーが何を担当しているか確認できます。

## 動作要件

- Home Assistant 2024.8 以降。
- HA を動かすホストが Bleak 対応プラットフォームであること
  （Linux / macOS / Windows）。**Android はプロトコルライブラリが
  非対応**のため、初期設定フローで明示的に拒否されます。
- 長距離用に、ロックの広告を *受信できる* BLE アダプターまたは
  `bluetooth_proxy` を 1 つ用意してください。
- 近距離用に、ロックに *接続できる*（目安 3 m 以内）BLE アダプター
  または `bluetooth_proxy` を 1 つ用意してください。長距離用と同じ
  アダプターでも構いません。

## セキュリティ

登録時に取得する `AdvDataKey` はロック固有の鍵で、状態広告を
復号できる唯一の手段です。暗証番号と同じ重要度の機密情報として
取り扱ってください。これらは HA の
`.storage/core.config_entries` 内の設定エントリに保存されます。
再登録するにはロックを再度登録モードに入れる必要があります。
